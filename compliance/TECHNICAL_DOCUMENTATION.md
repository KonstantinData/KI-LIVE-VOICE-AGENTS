# Technical Documentation - KI-LIVE-VOICE-AGENTS

**Legal basis:** EU AI Act technical documentation baseline  
**Status:** July 2026

## System Overview

This repository provides the live voice and chat runtime for tenant-specific AI
agents. For `mein-kuechenexperte`, the public agent is KEA.

## Components

| Component | Technology | Purpose |
| --- | --- | --- |
| Agent runtime | Python 3.12 | Prompt assembly, context handling, tool boundary enforcement |
| API | FastAPI | Widget, WebSocket, Live Voice, uploads, and CRM handoff endpoints |
| Database | PostgreSQL + pgvector | Runtime conversations, messages, upload metadata, and audit events |
| Widget | React/Vite IIFE | Embeddable customer chat and voice UI |
| LLM provider | OpenAI | Chat, Live Voice, summaries, and upload analysis |
| CRM integration | HTTPS webhooks | Sanitized contact and usage handoff to the external CRM repository |

This repository does not operate CRM screens, CRM dashboards, lead/contact CRUD,
or cost-report UI.

## Data Flows

```text
Website visitor browser
  -> widget / WebSocket / WebRTC
  -> FastAPI runtime on Hetzner
  -> OpenAI where enabled and consented
  -> PostgreSQL runtime database
  -> external CRM webhook for contact and usage handoff
```

Uploads are stored in private backend storage. Authorized CRM users access them
through short-lived signed URLs generated by the external CRM flow.

## Models

| Model | Provider | Purpose |
| --- | --- | --- |
| `gpt-realtime-2.1` | OpenAI | Live Voice sessions |
| `gpt-4o-mini` or configured chat model | OpenAI | Chat, upload summary, and supporting analysis |
| `text-embedding-3-small` | OpenAI | Knowledge search where enabled |

## Limits

- The agent does not collect contact details by voice.
- Contact data is submitted through a secure form and handed off to the external CRM.
- Local runtime memory stores conversation context, not CRM lead ownership data.
- No CRM dashboard or CRM record derivation is implemented in this repository.
